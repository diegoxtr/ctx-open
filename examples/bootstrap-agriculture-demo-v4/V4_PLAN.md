# Bootstrap Agriculture Demo V4 Plan

## Objective

Validate the first branch-like hypothesis layer on the same contradiction-heavy agricultural text used in prior bootstrap regression cases.

## Why v4 exists

`v3` proved:

- bootstrap can preserve multiple competing interpretations

`v4` should prove:

- those interpretations can now behave like branches inside CTX without becoming repository branches

## Source

Use the same contradiction-heavy agricultural article as the baseline comparison case.

## Expected behavior

After bootstrap and review, the repo should be able to express:

- branch state per hypothesis
- branch role per hypothesis
- shared lineage grouping
- explicit competing relations
- merge or supersede transitions
- evidence sharing across interpretations without forcing synthesis

## Pass condition

The experiment passes only if at least these interpretations can be represented as separate branch-like hypotheses inside one lineage group:

1. microbiota-first
2. microclimate-first
3. complex interaction / hidden-variable model

And at least these operations are validated:

1. set branch-state or branch-role
2. relate competing hypotheses
3. merge one hypothesis into another
4. share evidence across interpretations

## Failure condition

The experiment fails if the model still forces these outcomes:

- no way to distinguish competing interpretations beyond plain hypothesis state
- no explicit relation model between interpretations
- no visible record of merge or evidence sharing

## Operational rule

Do not treat repository branching as part of the pass condition.

`v4` is only about branch-like semantics inside hypothesis lineage.
