# CTX Project Philosophy
If a language model and its agent lose context, this is the tool you need.

CTX is built around a simple idea:

- structured reasoning should be versionable
- the cognitive repository should live locally
- the `.ctx` format must be portable
- the user should keep control of data and workflow

## Principles

### 1. Local-first usage

CTX must be installable, runnable, and usable locally without relying on a central service.

### 2. Modifiable and extensible

CTX should be modifiable, adaptable, and extensible. The project does not aim to prevent forks or changes; it aims to prevent improper appropriation of authorship, brand, and the official service.

### 3. Open standard

The `.ctx` format must be an open, portable standard. A cognitive repository should not be locked into a closed platform.

### 4. Commercializable software

CTX allows commercial local and on-premise use under the project license. The core business is software.

### 5. No competing service without a license

What the license restricts is not use or modification of the software, but offering CTX as a competing service without separate authorization.

### 6. Brand and official direction protected

The CTX name, official distribution, and project direction remain under control of the project holder.

## Practical model

CTX seeks this balance:

- open to use
- open to modify
- open to redistribute under the same license
- protected against commercial appropriation of the official service

## What CTX wants to enable

- local cognitive repositories
- real technical adoption
- extensions and integrations
- self-hosted deployments
- professional and commercial on-premise use

## What CTX wants to avoid

- forks presented as the official distribution
- operation as a competing SaaS without a license
- capture of the `.ctx` standard by a third party

## Strategic direction

CTX is not meant to be a closed tool. It is meant to be an open base for cognitive versioning, with a strong official project, a clear license, and consistent technical direction.
