# CTX - Release Interna 0.1.0
Si un modelo de lenguaje y su agente pierden el contexto, esta es la herramienta que necesitas.

## Estado de la release

Esta release interna establece una base funcional para validacion tecnica controlada.

Version:

- `0.1.0-internal`

Formato de repositorio:

- `1.0`

## Que incluye

- repositorio cognitivo local `.ctx/`
- modelo de dominio estructurado
- CLI operativa
- commits cognitivos reproducibles
- diffs cognitivos
- branching, checkout y merge
- deteccion de conflictos cognitivos
- context packets
- runs con providers
- metrics, packet inspection, run inspection y provider inspection
- tests automatizados
- documentacion base de plan, piloto e instalacion

## Para que sirve esta release

Esta release no apunta todavia a distribucion abierta. Su objetivo es:

- permitir pruebas tecnicas controladas
- consolidar una linea base estable
- medir utilidad real del producto
- detectar fricciones antes de cerrar V1

## Criterio de uso

La release puede usarse para:

- pilotos internos
- evaluacion tecnica
- demostraciones controladas
- experimentacion de flujo cognitivo estructurado

## Limites actuales

Todavia no debe considerarse:

- producto final
- V1 cerrada
- plataforma multiusuario
- solucion remota o enterprise

## Siguiente objetivo

El siguiente objetivo operativo es convertir esta release interna en una candidata seria para piloto V1 mediante:

- mejora de UX CLI
- resolucion guiada de conflictos
- endurecimiento de providers reales
- pruebas piloto con usuarios
