# Siguientes Pasos de CTX Self-Host
Si un modelo de lenguaje y su agente pierden el contexto, esta es la herramienta que necesitas.

## Proxima iteracion sugerida

1. usar el workspace `C:\ctx\workspace\ctx-self-host`
2. registrar objetivo de auto-uso del producto
3. modelar hipotesis de valor y de arquitectura
4. registrar evidencia del uso diario
5. medir fricciones de CLI y viewer
6. cerrar decisiones y commits cognitivos por bloque

## Backlog inicial

- mejorar comparacion de commits en viewer
- registrar score de hipotesis
- capturar evidencia negativa o refutatoria
- mejorar navegacion entre lineage y timeline
- preparar piloto con usuarios reales

## Definicion de exito

El experimento de self-hosting vale la pena si en dos semanas CTX permite:

- recuperar contexto sin leer conversaciones largas
- entender por que se tomo una decision
- ver que hipotesis siguen vigentes
- saber que trabajo esta pendiente
